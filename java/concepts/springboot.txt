Springboot
==========
Spring Boot is an extension of the Spring framework that simplifies application development by providing auto-configuration, embedded servers, and starter dependencies

Need
----
Spring Boot was introduced to reduce boilerplate configuration, simplify dependency management, and enable rapid development using auto-configuration and embedded servers.

Problem Solved by Springboot:
----------------------------
1.Auto Configuration : you just add dependency, SB will automatically do the setup
2.Starter dependencies
3.Embedded servers : no need for external tomcat
4.Minimal Configuration
5.Production ready features: acutators

Feature			Spring		Spring Boot
Config			Manual		Auto
Server			External	Embedded
Setup time		High		Low
Dependencies	Manual		Starter
XML				Required 	Not needed

Confitional Annotations:
-----------------------
1.@ConditionalOnClass: if present in classpath then enable config
ex: if DataSource dependency is present then enable the bean otherwise skip

@Configuration
@ConditionalOnClass(name = "javax.sql.DataSource")
class DBConfig {

    @Bean
    public MyDBService dbService() {
        return new MyDBService();
    }
}

2.@ConditionalOnMissingBean: If bean is not present then create it otherwise skip

@Bean
@ConditionalOnMissingBean(BClass.class)
public AClass aClass() {
System.out.println("AClass is created");
return new AClass();
}

@Bean
public BClass bClass() {
System.out.println("BClass is created");
return new BClass();

In this example, AClass will only be created if there is no bean of type BClass in the Spring context. If BClass is already present, AClass will not be created.

Auto-Configuration:
------------------
Auto-configuration automatically configures Spring beans based on the classpath, existing beans, and application properties

ex: we add this dependency:
spring-boot-starter-web
springboot internally setup : DispatcherServlet, Tomcat, Jackson, MVC config

"Spring Boot auto-configuration works by scanning the classpath and loading configuration classes conditionally using annotations like @ConditionalOnClass and @ConditionalOnMissingBean. It automatically configures beans based on available dependencies."

REST APIs:
---------
REST: Representational State transfer
REST APIs allow communication between client and server using HTTP methods like GET, POST, PUT, DELETE

Request Handling:
----------------
1.Path Variables:
@GetMapping("/users/{id}")
public String getUser(@PathVariable int id) {
    return "User " + id;
}

2.Query Param:
@GetMapping("/users")
public String getUser(@RequestParam String name) {
    return name;
}

3.Request Body : JSON to Java object converter (Jackson)
@PostMapping("/users")
public String create(@RequestBody User user) {
    return user.getName();
}

4.Response Handling:
return ResponseEntity.status(201).body(UserDto);

Exception Handling:
------------------
problem:
@GetMapping("/user/{id}")
public User getUser(@PathVariable Long id) {
    return repo.findById(id).get(); // 💀 crash if not found
}

Solutions:
1.Custom Exception:
class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String msg) {
        super(msg);
    }
}

2.Throw Exception
public User getUser(Long id) {
    return repo.findById(id)
        .orElseThrow(() -> new UserNotFoundException("User not found"));
}

3.Global Exception Handler
@ControllerAdvice
class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<String> handleUserNotFound(UserNotFoundException ex) {
        return ResponseEntity.status(404).body(ex.getMessage());
    }
}

Limitations of Springboot:
-------------------------
1. Because of Auto-Configuration it is difficult to debug
2. Less control over abstraction
3. High startup time
4. huge deployment size due to fat jar
5. Version conflicts

Spring Boot simplifies development but introduces challenges like reduced control due to auto-configuration, higher memory usage, slower startup time for large applications, and debugging complexity due to hidden configurations
